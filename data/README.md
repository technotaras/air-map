## Структура

Каждая из 22 пронумерованных папок (`01/` … `22/`) содержит:

- `comment.txt` — текстовые заметки из Notion (пустой файл, если в Notion не было заметок)
- `video.mp4` — видеофайл (**не включён**, см. ниже)

Дополнительно:

- `manifest.json` — сводный список всех записей со ссылками на attachment'ы Notion и путями к файлам

## Записи без комментариев

В оригинале пустые комментарии у записей **12, 13, 18** — их `comment.txt` пустые, как и на странице Notion.

## Видеофайлы

Видео в Notion хранятся как приватные attachment'ы и не скачиваются через публичный API / MCP. Чтобы наполнить папки видео:

1. Откройте страницу в десктопном или веб-клиенте Notion
2. Для каждого видео: правый клик → **Download** (или скачайте через меню **...**)
3. Переименуйте скачанный файл в `video.mp4` и положите в соответствующую папку

Соответствие номера папки → attachment / оригинального имени файла — в `manifest.json` (поля `video_attachment_id` и `video_original_filename`).

## Для последующей разработки

В `manifest.json` уже есть машиночитаемая структура: `notion_block_id`, ID attachment'а, путь к видеофайлу и путь к комментарию. Парсер проекта может пройти по `entries[]` и для каждой записи прочитать `comment.txt`, а при наличии `video.mp4` — использовать его.

Внутри комментариев (см. `*/comment.txt`) есть полезные структурированные поля, которые можно распарсить:

- Координаты MGRS (`36T US ...` / `36T TS ...`)
- Тип цели (Гербера / Шахед / Орлан)
- Высота, дистанция, азимут
- Номер цели, номер по Скаймапі
- Экипаж, подразделение, дата/время взлёта и подрыва
- Засіб / борт (Тарас+)

Формат записей в Notion не унифицирован (разные списки, часть полей отсутствует), поэтому парсер должен быть толерантен к отсутствию полей.
